> 说明：以下建议仅面向虚构推演、教学展示和交互设计优化，不涉及真实作战指导。

## 以“资深军事推演用户”视角给出的总体评价

当前版本已经具备较强的“态势可视化演示能力”，尤其在以下方面表现不错：

- 二维 / 三维切换直观
- 地图交互顺手，右键编辑效率高
- 命令线、区域标绘、探测圈已经形成基础推演表达能力
- 数据服务、态势展示、知识图谱之间有初步闭环

但如果从“更像一个成熟的军事推演沙盘”角度看，当前仍偏重“静态展示”，还缺少：

- 时间维度
- 不确定性表达
- 指挥流程反馈
- 编组层级与任务目标
- 推演结束后的复盘机制

下面按优先级给出建议。

## P0：优先级最高

- [ ] 增加时间轴与态势回放
  - 价值：这是从“静态沙盘”升级到“推演系统”的第一步。
  - 建议：支持要素按时间片出现、移动、隐藏、变色和注记变化。

- [ ] 给探测圈加入“探测结果状态”而不只是“几何范围”
  - 价值：真正的传感器展示，不应只有覆盖范围，还要体现“发现 / 疑似 / 丢失 / 持续跟踪”等状态。
  - 建议：目标点可附带状态标签、置信度条、更新时间和来源传感器。

- [ ] 引入任务目标与裁决条件
  - 价值：当前更像编辑器，缺少“这一局要干什么”。
  - 建议：为场景增加目标点、关键区域、阶段目标和完成状态。

- [ ] 增加复盘视图
  - 价值：推演系统的核心价值之一不是“演”，而是“演完之后能复盘”。
  - 建议：记录关键节点、命令变化、探测变化、环境变化，并生成时间线摘要。

## P1：高优先级

- [ ] 增加单位编组层级
  - 价值：现实中的态势展示通常不只关心单个平台，更关心编组、分队、梯队与任务包。
  - 建议：支持单位归属“分队 / 分组 / 编队 / 批次”，并支持整组显隐、整组拖动、整组过滤。

- [ ] 增加环境对感知和机动的影响表达
  - 价值：当前环境图层已经有雏形，但还没有真正影响态势表现。
  - 建议：让天气、地形、电磁环境影响探测圈透明度、单位颜色提示或行动状态标签。

- [ ] 增加“信息时效性”表达
  - 价值：指挥态势里“这条信息是不是刚更新的”非常关键。
  - 建议：每个态势要素显示最后更新时间、来源、可信度、是否过期。

- [ ] 增加简化的保障状态
  - 价值：如果只有位置，没有状态，就很难支撑更深层的推演体验。
  - 建议：增加油料、弹药、通信、健康、完好率等抽象状态条，但保持演示级复杂度。

## P2：中优先级

- [ ] 增加多套想定版本管理
  - 价值：推演经常需要对比方案 A / B / C，而不是只保留一个当前态势。
  - 建议：支持场景快照、版本对比、注释和回滚。

- [ ] 增加协同编辑与标注
  - 价值：多人共同标绘和批注，能明显提升演示价值和评审效率。
  - 建议：支持“谁在编辑什么”“批注未读”“锁定元素”等协同机制。

- [ ] 增加地理测量和吸附能力
  - 价值：当标绘更复杂时，精度和效率会很重要。
  - 建议：增加顶点吸附、线面吸附、长度 / 面积量算、方向角提示。

- [ ] 增加更完整的传感器体系
  - 价值：现在已支持雷达、红外、光电、声学四类探测圈，是非常好的起点。
  - 建议：下一步可以加入“搜索模式 / 跟踪模式 / 被动模式 / 扫描周期 / 刷新率”等轻量属性。

## UI / 交互层建议

- [ ] 给探测圈增加图例说明
  - 建议：在侧栏中展示四类传感器的颜色、包络线含义和典型用途。

- [ ] 给二维 / 三维切换增加状态提示
  - 建议：明确告知二维模式已自动关闭 DEM，避免用户误判为位置异常。

- [ ] 增加默认演示场景按钮
  - 建议：提供“一键生成典型演示场景”，让首次体验更快进入状态。

- [ ] 增加新手引导
  - 建议：首次进入时，用 4~5 步提示说明拖放、右键、编辑、导出、视图切换。

## 数据 / 架构层建议

- [ ] 为态势实体增加 schema 校验
  - 建议：尤其是坐标、半径、传感器类型、俯仰范围、方位范围，避免脏数据导致渲染异常。

- [ ] 区分“模板实体”和“实例实体”
  - 建议：模板负责预设视觉样式和参数，实例只保存实际坐标与状态。

- [ ] 增加导入 / 导出场景包
  - 建议：把态势实体、环境、说明、缩略图打包，便于演示复用。

## 推荐近期实施顺序

1. 时间轴与回放
2. 探测结果状态与置信度
3. 任务目标 / 胜负条件
4. 复盘视图
5. 编组层级
6. 环境影响联动
7. 协同编辑
8. 场景包导入导出
