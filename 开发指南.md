# 开发指南

> 本文件面向后续开发者和接手本项目的 agent，说明代码文件分布、职责边界和常见修改入口。开始开发前请先读取 `AGENTS.md`、`agent.md`、`README.md` 和本文件。

## 1. 快速定位

- `AGENTS.md`：协作规则和必读顺序。任何 agent 接手项目时先看这里。
- `agent.md`：技术交接记忆。记录最近修改、验证结果、剩余风险。
- `README.md`：用户侧说明。记录当前真实功能、启动方式、已知行为。
- `开发指南.md`：开发侧文件地图。说明各文件实现什么功能、维护时该改哪里。
- `TODO.md`：待办和后续优化建议。

## 2. 根目录文件

- `package.json`：根工作区脚本入口。负责同时启动前后端、构建、生产启动、瓦片脚本代理。
- `package-lock.json`：根依赖锁文件。依赖变化后应同步提交。
- `.gitignore`：Git 忽略策略。当前明确忽略前端构建产物、依赖、本地地图大目录、运行数据库和本地密钥。
- `logo.png`：全站品牌 logo 源文件，由 `apps/web/src/config/branding.js` 引用。
- `start-mac.sh`：macOS 启停脚本，支持 `dev / web / server / prod / status / stop / restart / check`。
- `start-*.bat`：Windows 一键启动脚本，分别覆盖开发、后端、前端和生产模式。
- `scripts/start-production.mjs`：生产模式启动器。缺少前端构建产物时自动构建 `apps/web/dist/client`，再启动后端。
- `scripts/generate-sample-tiles.ps1`：生成示例离线瓦片到 `apps/web/tiles`。
- `scripts/download-sample-tiles.mjs`：下载或准备示例瓦片资源到 `apps/web/tiles`。

## 3. 前端入口与全局配置

- `apps/web/package.json`：前端包配置，定义 `vite` 开发、构建和预览脚本。
- `apps/web/vite.config.js`：主 Vite 配置。接入 Vue、Cesium 静态资源插件、`/api` 代理、构建输出目录，并在开发模式把 `apps/web/terrain`、`apps/web/dem`、`apps/web/tiles` 分别挂到 `/terrain`、`/dem`、`/tiles`；兼容历史 `public` 和误拼写 `pubulic` 本地资源目录；请求本地 `/terrain/layer.json` 或 `/dem/layer.json` 时会调用后端共享的 `local-terrain-layer.js` 动态修复 Cesium terrain 可用性元数据。
- `apps/web/vite.config.auth-check.js`：认证检查相关的辅助 Vite 配置。
- `apps/web/index.html`：前端 HTML 入口。
- `apps/web/src/main.js`：Vue 应用启动入口，挂载根组件和路由。
- `apps/web/src/App.vue`：根组件，承载路由出口。
- `apps/web/src/router/index.js`：前端路由表。维护首页、登录、任务中心、数据服务、能力/行动/消耗和规划模块路由。
- `apps/web/src/styles.css`：全局样式和模块样式集中地。大部分页面布局、卡片、工作台、三维面板和响应式规则都在这里。
- `apps/web/src/config/branding.js`：品牌名称和 logo URL 配置。`PageBrand.vue` 会读取这里显示统一品牌。
- `apps/web/src/api.js`：前端 API 客户端。封装请求、错误归一、NDJSON 流式请求、各业务接口。
- `apps/web/src/auth.js`：登录态管理。负责登录、登出、当前用户状态和鉴权请求。
- `apps/web/src/utils/exporters.js`：导出工具。负责三维视图截图、PDF、KML 等导出辅助能力。
- `apps/web/src/data/situationCatalog.js`：前端态势要素和演示目录数据，属于源码数据，已明确允许 Git 跟踪。

## 9. 后端入口与服务文件

- `apps/server/package.json`：后端包配置，定义开发、启动、测试脚本。
- `apps/server/src/index.js`：Express 服务入口。负责鉴权、中间件、API 路由、静态前端托管、`apps/web` 离线地图资源挂载、离线 terrain `layer.json` 修复和规划流式执行端点。
- `apps/server/src/local-terrain-layer.js`：本地 Cesium terrain 元数据修复工具。根据真实 `.terrain` 文件生成 `available` 范围、过滤越界瓦片，并移除会让 Cesium 忽略顶层可用性的 `metadataAvailability`。
- `apps/server/src/local-terrain-layer.test.js`：本地 terrain 元数据修复回归测试。
- `apps/server/src/db.js`：SQLite 数据层。负责建表、种子数据、任务、资源、导入、执行记录和结果读写。
- `apps/server/src/import-preview.js`：文件导入预览解析。支持 Word、PDF、Excel、CSV、TXT 等格式并生成可读预览和抽取草稿。
- `apps/server/src/algorithm-gateway.js`：外部算法服务统一网关。负责请求结构、超时、错误归一和调用日志。
- `apps/server/src/capability.js`：能力评估后端算法与模板。
- `apps/server/src/action.js`：行动计算后端算法与模板。
- `apps/server/src/consumption.js`：消耗计算后端算法与模板。
- `apps/server/src/planning.js`：规划模块模板、接口辅助和规划相关服务逻辑。
- `apps/server/src/planning-runtime.js`：智能任务规划核心运行时。实现六类内置规划算法、任务流程执行、依赖校验、结果打包和进度事件。
- `apps/server/src/planning-threat-python.js`：Python 威胁分析适配器。负责整理输入文档、注入大模型配置、启动 Python 子进程、读取 stderr 流和返回结构化结果。
- `apps/server/src/planning-airlanding-python.js`：Python 智能机降算法适配器。负责定位系统离线 Cesium terrain、启动 `airlanding_zone` 子进程、转发 stderr 流和返回机降地域结果。
- `apps/server/src/*.test.js`：Node 内置测试。覆盖导入预览、规划运行时和 API 契约。

## 10. Python 规划算法文件

- `apps/server/planning-python/requirements.txt`：Python 依赖清单。
- `apps/server/planning-python/theat_analyze/analyze.py`：第一阶段威胁分析入口。执行文档解析、LLM 结构化抽取、威胁场计算和热力图生成。
- `apps/server/planning-python/theat_analyze/extractor.py`：文档解析和 LLM 流式抽取逻辑。负责 OpenAI 兼容接口 / Ollama 后端、JSON 修复和态势结构校验。
- `apps/server/planning-python/theat_analyze/generate_assessment.py`：第二阶段敌情研判入口。基于第一阶段结果生成作战意图、部署态势和报告。
- `apps/server/planning-python/theat_analyze/assessment_report.py`：研判报告生成逻辑。
- `apps/server/planning-python/theat_analyze/threat_analyzer.py`：威胁分析核心计算函数。
- `apps/server/planning-python/theat_analyze/geo_math.py`：地理计算辅助函数。
- `apps/server/planning-python/theat_analyze/schemas.py`：Python 侧数据结构和校验模型。
- `apps/server/planning-python/theat_analyze/api_server.py`：Python 算法独立 API 服务入口，当前 Node 主链路优先使用子进程适配器。
- `apps/server/planning-python/airlanding_zone/main.py`：智能机降算法入口。读取 Node 生成的 JSON 输入，组织机降需求、候选区生成、分组选优和 JSON 输出。
- `apps/server/planning-python/airlanding_zone/candidate_generator.py`：机降候选地域生成核心。根据威胁目标、搜索距离、地形采样、坡度/起伏约束和地表风险生成候选多边形。
- `apps/server/planning-python/airlanding_zone/dem_provider.py`：地形采样 provider。当前默认读取系统 Cesium `quantized-mesh-1.0` terrain 瓦片，也保留 GeoTIFF/OpenTopography 兼容 provider。
- `apps/server/planning-python/airlanding_zone/optimizer.py`：候选机降区排序、间距约束和分组优选逻辑。
- `apps/server/planning-python/airlanding_zone/threat_field.py`：复用 `theat_analyze` 威胁场计算能力，对候选点进行威胁度评估。
- `apps/server/planning-python/airlanding_zone/config.py`：智能机降算法配置。默认 terrain 根目录为 `apps/web/terrain`，并兼容 `apps/web/pubulic/terrain`、`apps/web/public/terrain`。
- `apps/server/planning-python/airlanding_zone/landcover_provider.py`：可选 OSM 地表风险采样，默认关闭联网查询。

## 12. 常见修改入口

- 修改品牌名或 logo：改 `apps/web/src/config/branding.js` 和根目录 `logo.png`。
- 修改全局视觉风格：优先改 `apps/web/src/styles.css`，再看具体组件结构。
- 新增前端页面：在 `apps/web/src/views/` 添加页面，并在 `apps/web/src/router/index.js` 注册路由。
- 新增通用组件：放入 `apps/web/src/components/`，避免把大块 UI 直接写进页面。
- 新增前端业务状态：放入 `apps/web/src/modules/`，页面只消费组合好的状态和动作。
- 新增后端 API：优先在 `apps/server/src/index.js` 接路由，数据读写放入 `db.js`，复杂算法放到对应业务文件。
- 修改规划算法：优先看 `apps/server/src/planning-runtime.js`；大模型威胁分析链路同时看 `planning-threat-python.js` 和 `apps/server/planning-python/theat_analyze/`；智能机降链路同时看 `planning-airlanding-python.js` 和 `apps/server/planning-python/airlanding_zone/`。
- 修改导入解析：看 `apps/server/src/import-preview.js`，并同步补 `apps/server/src/import-preview.test.js`。
- 修改启动方式：看根 `package.json`、`start-mac.sh`、`start-*.bat` 和 `scripts/start-production.mjs`。
- 修改离线地图资源路径、terrain 元数据修复、回退策略或三维地形渲染开关：看 `apps/server/src/local-terrain-layer.js`、`apps/web/vite.config.js`、`apps/server/src/index.js`、`apps/web/src/components/CesiumGlobe.vue`、`apps/web/src/components/SituationWorkbench.vue` 和 `apps/web/src/components/ResourceWorkbench.vue`。

## 13. 验证建议

- 前端构建：`npm run build --workspace @mission/web`
- 后端测试：`npm test --workspace @mission/server`
- 后端语法检查：`node --check apps/server/src/index.js`
- 离线 terrain 修复检查：`node --test apps/server/src/local-terrain-layer.test.js`
- 前端模块语法检查：`node --check apps/web/src/api.js` 或对应 JS 模块
- Python 语法检查：`python3 -m py_compile apps/server/planning-python/theat_analyze/*.py apps/server/planning-python/airlanding_zone/*.py`
- Git 忽略验证：`git check-ignore -v apps/web/terrain/layer.json apps/server/data/mission-demo.sqlite apps/.env.local`

## 14. 维护约定

- 修改代码后必须同步更新 `agent.md` 和 `README.md`；如果涉及文件职责、结构或入口，也要更新本 `开发指南.md`。
- 不要提交 `apps/web/node_modules/`、`apps/web/dist/`、`apps/web/terrain|dem|tiles|public|pubulic`、`apps/server/data/`、`node_modules/`、`.env.local` 或 Python 生成报告。
- 大模型 API Key 和 Base URL 不应写死在源码中，应通过前端配置或服务端环境变量注入。
- `agent.md` 记录技术交接事实，`README.md` 面向使用者，`开发指南.md` 面向开发者。
