# 开发技术文档

## 适用范围

本文档面向当前任务规划模块中的 6 个内置算法：

1. 敌情威胁自动分析
2. 作战力量智能编组
3. 作战目标自动分配
4. 作战方法自动规划
5. 作战保障自动规划
6. 机降地域优化选择

目标是帮助开发阶段快速明确每个算法的输入、输出和实现思路，并梳理它们之间的上下游依赖关系。

## 总体串联关系

建议把 6 个算法理解为一条可分阶段联调的流水线：

1. `敌情威胁自动分析`
   负责把多源资料转成结构化敌情、威胁场和目标线索。
2. `作战力量智能编组`
   基于我方兵力、约束和敌情结果，生成编组方案。
3. `作战目标自动分配`
   基于敌情目标与我方编组，完成平台级目标分配。
4. `作战方法自动规划`
   基于目标分配、地形环境和威胁约束，生成航路/行动路线。
5. `作战保障自动规划`
   基于编组、路线、战损预测和资源池，生成保障调度方案。
6. `机降地域优化选择`
   在机降类任务中，为路线与保障提供候选机降点和优选地域。

其中：

- `敌情威胁自动分析` 是大多数后续算法的基础输入源。
- `作战力量智能编组` 和 `机降地域优化选择` 都可能被后续算法引用。
- `作战保障自动规划` 对上游依赖最强，通常需要编组、路线，机降任务下还需要机降地域结果。

## 算法一：敌情威胁自动分析

### 输入

- 资源库中显式勾选的数据源
- 本地上传文件：`Word / PDF / Excel / CSV / TXT`
- 已抽取的情报、环境、态势文本
- 可选的地图/地理范围信息
- 可选的算法参数（暂时未实现）：
  - 分析模式
  - 阈值
  - 分类标签
  - 结果过滤条件

### 输出

- `targetEntities`
  结构化敌方目标列表，如目标类型、位置、重要度、威胁等级（暂时只规定这些）
- `heatmapBase64` / `heatmapGeojson`
  可视化威胁热区（已实现）
- `bounds`
  热力图叠加范围（已实现）
- `pointThreatEvaluation`
  采样点威胁值或关键点威胁评估（已实现）
- `situationMap`
  部署区、火力覆盖、预警圈、方向轴等结构化态势图层
- `assessmentReport`
  面向展示的综合分析结论（已实现）
- `evidenceTrace`
  **证据溯源信息，指向源文件、抽取片段和来源类型（已实现）**

### 大致思路
拆成 4 层：

1. 输入整理层
   把资源库文本、上传文件、环境记录统一整理成标准输入包。
2. 信息抽取层
   从非结构化文本中抽取目标、装备、阵地、区域、时间、能力和关系，转化成结构化json文件。
3. 威胁建模层
   按目标类型、射程、部署密度、覆盖范围和关键区域计算威胁场。
4. 结果组装层
   同时输出结构化目标、三维叠加图层、热力图和自然语言报告。

### 实现路径

- 第一版可先做“规则抽取 + 模板化评分”。
- 如果已有 Python 或大模型抽取链路，可保留为优先路径，但必须有规则兜底。
- 输出字段要稳定，尤其是 `targetEntities`、`heatmapGeojson`、`bounds`，因为后续算法会直接依赖。

## 算法二：作战力量智能编组

### 输入

- 我方兵力清单
  - 平台类型
  - 数量
  - 载荷/武器
  - 航程/续航
  - 角色标签（文档格式，使用llm进行解析）


- 资源库中的我方兵力资料和本地上传兵力文件
- 敌情威胁分析结果
  - 目标类型
  - 威胁方向
  - 任务区域
- 任务模板约束
  - 火力打击 / 机降突击（投放兵力，靠近目标点）等任务类型
- 编组参数
  - `expectedGroupCount`
  - 约束模型
  - 偏好权重

1.考虑编制，2.考虑兵力
### 输出

- 编组方案列表
- 每个方案下的群组明细
  - 群组编号
  - 平台构成
  - 任务角色
  - 载荷与能力摘要
- `actualGroupCount`
- 规则权重和约束评分
- 迭代优化信息
- 推荐方案与推荐理由
- `evidenceTrace`

### 大致思路

建议按“约束满足 + 多目标优化”的思路做：

1. 兵力标准化
   把结构化兵力和文档抽取兵力统一成平台池。
2. 任务画像生成
   根据敌情结果形成需求画像，如压制、防空、侦察、突击、运输。
3. 候选编组生成
   先按角色和约束快速拼出若干初始群组。
4. 方案优化与排序
   综合火力覆盖、任务完整性、平台均衡、风险暴露和资源利用率打分。

### 实现建议

- 第一阶段可先做启发式编组：按角色模板分桶，再局部调优。
- 第二阶段再引入遗传算法、模拟退火或整数规划。
- 输出中最好保留“为什么这么编”，否则用户很难信任结果。

## 算法三：作战目标自动分配

### 输入

- 敌情威胁分析输出的目标列表
- 作战力量编组结果
- 平台能力参数
  - 射程
  - 弹药/载荷
  - 目标适配类型
  - 同时攻击能力
- 任务规则
  - 波次约束
  - 协同约束
  - 覆盖优先级
- 分配算法选择
  - 匈牙利算法
  - 蚁群协同
  - 多目标优化

### 输出

- 平台级目标分配结果
- 波次分配结果
- 未分配目标列表
- 超载/冲突说明
- 方案对比表
- 合理性校核结果
- 目标覆盖摘要
- 调整建议

### 大致思路

核心是把“平台打哪个目标”建模为匹配或优化问题：

1. 生成收益矩阵
   平台对目标的收益可由目标重要性、命中适配度、射程利用率、风险代价共同决定。
2. 施加硬约束
   如平台容量、波次数量、禁配关系、最小覆盖要求。
3. 运行分配算法
   单波次可先用匈牙利算法，多波次和协同时再扩展到启发式优化。
4. 校核与解释
   输出为什么某些目标未分配、为什么某平台承担高价值目标。

### 实现建议

- 先从单波次、单平台单目标模型起步，保证链路稳定。
- 再逐步扩展到多波次、协同打击、备选重分配。
- 输出里要保留成本矩阵或评分依据，方便后续调试。

## 算法四：作战方法自动规划

### 输入

综合考虑空间路径和时间

- 作战目标分配结果
- 敌情威胁场结果
- 地形、气象、电磁等环境约束
- 我方平台机动参数
  - 速度
  - 航程
  - 最小转弯半径
  - 飞行高度范围
- 起点、集结点、目标点、返航点等锚点
- 路径算法选择
  - `A*`
  - `Dijkstra`
  - `RRT`

### 输出

- 每条路线的路径点序列
- 波次、平台、目标对应关系
- 检查点与时间窗
- 路径代价摘要
  - 距离
  - 时间
  - 威胁穿越代价
  - 地形/环境代价
- 三维叠加图层
- 备选路线与推荐路线

### 大致思路

这是一个典型的“带约束路径规划”问题：

1. 建图
   基于栅格、采样点或导航图构建搜索空间。
2. 代价场融合
   把威胁热区、地形障碍、气象、电磁暴露统一映射到代价场。
3. 路径搜索
   A* 适合确定性和较快求解，RRT 适合复杂连续空间。
4. 路线后处理
   对原始路径做平滑、检查点提取、时间窗估计和可飞行性校验。

### 实现建议

- 第一版可以用二维栅格代价场，先打通流程。
- 三维高度层可作为第二阶段扩展。
- 如果上游目标分配为空，建议回退到目标锚点生成基础路线，避免整条链路中断。

## 算法五：作战保障自动规划

### 输入

规划完毕之后，计算保障点

- 作战力量编组结果
- 作战方法规划结果
- 机降任务下的机降地域结果
- 战损预测输入
  - 装备损失率
  - 人员伤亡率
  - 受损装备数
  - 伤员数
  - 关键窗口数
- 保障资源池
  - 弹药
  - 油料
  - 维修
  - 医疗
  - 空域
  - 通信
- 投送与节点能力
  - 运输架次
  - 单架次载重
  - 维修分队
  - 医疗分队
  - 协同席位
  - 调度链路容量

### 输出

- 分阶段保障计划
- 资源调度结果
- 各类资源消耗与补给明细
- 节点负荷情况
- 瓶颈识别结果
- 任务匹配分析
- 保障风险提示
- 调整建议

### 大致思路

可以把它看成一个“需求预测 + 资源调度”问题：

1. 需求测算
   根据编组规模、路线长度、战损预测和任务窗口估算资源需求。
2. 资源匹配
   用库存、运输能力和节点能力对需求进行可行性校核。
3. 调度求解
   形成补给批次、配送顺序、维修优先级、救治分流方案。
4. 瓶颈分析
   标出哪类资源不足、哪个节点过载、哪个时间窗最紧张。

### 实现建议

- 第一版不一定要上复杂线性规划，可以先做规则驱动调度。
- 但要强制输出“短缺资源”和“不可满足约束”，不能只给一个看起来完整的计划。
- 对缺上游结果的情况要直接报错，避免生成误导性的默认保障方案。

## 算法六：机降地域优化选择

### 输入

- 候选机降区域或候选点
- 地形数据
  - 高程
  - 坡度
  - 起伏
- 敌情威胁场
  - 防空火力
  - 侦察覆盖
  - 热区分布
- 任务目标锚点
- 直升机/机降平台参数（***这个暂时还未考虑好怎么加入，影响什么情况呢？或者直接设置一个区域面积，通过这些参数，计算所需要的区域面积，然后再选点***）
  - 起降条件
  - 机动半径
  - 运载能力
- 评估权重
  - 安全性
  - 接近性
  - 隐蔽性
  - 投送效率

### 输出

- 候选机降点/区域评分表
- 最优机降点排序
- 每个候选点的优缺点解释（这里放弃接入大模型，而是从多指标进行评判，输出为：xxx指标评分，最后使用llm进行一个总评）
- 风险提示
- 地图标注结果
- 与后续路线/保障的联动建议

### 大致思路

本质上是一个多指标选址问题：

1. 候选生成
   先从地形或人工输入中生成候选点。
2. 可用性过滤
   去掉坡度过大、威胁过高、距离不合理的候选点。
3. 多指标评分
   从安全、距离、接敌效率、撤离便利、保障可达性等维度打分。
4. 排序与解释
   输出推荐点以及不推荐其他点的原因。

### 实现建议

- 第一版可先做加权评分模型。
- 后续再加入更细粒度地形遮蔽、飞行走廊和机降后扩散空间评估。
- 输出最好同时给出“最优点”和“备选点”，方便人工切换。

## 推荐的数据契约

为了让 6 个算法能稳定串联，后期统一采用以下结构：

```json
{
  "algorithm": "enemy-threat-analysis",
  "status": "implemented",
  "summary": "摘要文本",
  "structuredOutput": {},
  "evidenceTrace": [],
  "metrics": {},
  "artifacts": []
}
```

关键建议：

- `structuredOutput` 放核心结果，供后续算法消费
- `summary` 放人读摘要
- `metrics` 放打分、耗时、规模统计
- `artifacts` 放 `GeoJSON / CSV / HTML` 等导出物
- `evidenceTrace` 放来源追溯
