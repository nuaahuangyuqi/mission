# T01 Bug 台账（S1/S2 基线）

更新时间：2026-04-01  
范围：规划、能力计算、权限鉴权、导入接口、结果页与共享任务基线。

## 分级规则

- `S1`：阻断主流程、权限边界或数据可信性，必须先收口再继续新需求开发。
- `S2`：不直接阻断主流程，但会显著影响可用性、可维护性或稳定性，应纳入当前迭代。

## S1/S2 收口顺序建议

1. 先收口 `BUG-S1-001 ~ BUG-S1-004`（权限边界和会话安全）。
2. 再处理 `BUG-S2-001 ~ BUG-S2-004`（可用性、稳定性、追溯性）。
